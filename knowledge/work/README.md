# Work

Project-specific docs organized by domain. Create subfolders as your projects evolve.

Suggested starting structure:

```
work/
├── prds/          Specs and PRDs (by year or cycle)
├── runbooks/      How-to guides, tool setup
├── reference/     Links, PDFs, tool notes
└── templates/     Project-specific templates
```

Adapt to your own domains and projects.
